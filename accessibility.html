Webtilgjengelighet

Webtilgjengelighet sikrer at alle, inkludert personer med nedsatt funksjonsevne, kan bruke nettsider. Dette prosjektet har fokusert på å oppfylle standardene fra WCAG 2.1, med særlig oppmerksomhet på kontrastforhold, alternativ tekst for bilder og navigasjon med tastatur.

Testresultater

Vi testet nettstedet ved hjelp av både automatiske og manuelle metoder. Her er resultatene:

  • Lighthouse-verktøyet: Scoret 100 i tilgjengelighet på både mobil og skrivebord.
  • Wave-verktøyet: Ingen store problemer funnet.
  • Manuelle tester: Bekreftet at siden kan navigeres ved hjelp av tastatur.

Resultater fra mobil og skrivebord

Kodingvalg for universell utforming

Vi har prioritert universell utforming for å sikre at nettstedet vårt er tilgjengelig for alle brukere, inkludert personer med funksjonsnedsettelser. Her er en detaljert gjennomgang av de viktigste kodingsvalgene og hvorfor de ble tatt.

  • Alt-tekst for bilder: Alt-tekster gir viktig kontekst til skjermleserbrukere som ikke kan se bildene. Vi har inkludert beskrivende alt-tekster for alle bilder, som: 
    <img src="images/Home_Accessibility_mobile.png" alt="Tilgjengelighetsscore for mobil - Home side">
  • Semantisk HTML-struktur: Ved å bruke semantiske HTML-elementer som <header>, <nav>, <main>, og <footer>, skaper vi en logisk struktur for innholdet. For eksempel: 
    
<header>
    <nav>
        <ul>
            <li><a href="index.html">Home</a></li>
        </ul>
    </nav>
</header>
  • Fargekontrast og lesbarhet: Kontrast mellom tekst og bakgrunn er nøye evaluert for å oppfylle WCAG 2.1-retningslinjene. For eksempel har vi brukt mørk tekst på lys bakgrunn for optimal lesbarhet: 
    
body {
    color: #333;
    background-color: #f9f9f9;
}
  • Tastaturnavigasjon: Alle interaktive elementer, som lenker og knapper, kan nås og brukes med tastatur. Dette ble testet manuelt ved hjelp av Tab-tasten. Vi har også sørget for at fokusindikatorer er synlige: 
    
a:focus {
    outline: 2px solid #004a99;
}
  • Responsivt design: Vi har brukt media queries for å optimalisere nettstedet for ulike skjermstørrelser, fra mobil til skrivebord: 
    
@media (max-width: 768px) {
    img {
        width: 100%;
        height: auto;
    }
}
  • Automatiske og manuelle tester: Vi brukte Lighthouse for å identifisere potensielle tilgjengelighetsproblemer. Selv om automatiske tester er nyttige, har vi også gjennomført manuelle sjekker for å sikre riktig tabbingsrekkefølge, logisk DOM-struktur og synlige fokusindikatorer.
  • Universell utforming som en helhet: Tilgjengelighet handler ikke bare om å oppfylle tekniske krav, men også om å skape en positiv brukeropplevelse. Gjennom prosjektet har vi fokusert på å gjøre nettstedet intuitivt, enkelt og meningsfylt for alle brukere, inkludert de med ulike funksjonshemninger.

Manuelle sjekker

Selv om automatiserte tester er nyttige, gjennomførte vi også manuelle tester for å verifisere følgende:

  • Riktig tabbingsrekkefølge for alle interaktive elementer.
  • Visuell rekkefølge følger DOM-rekkefølge.
  • Fokusindikatorer er synlige og lett gjenkjennelige.
